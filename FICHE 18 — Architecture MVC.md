# FICHE 18 — Architecture MVC

---

## Problématique

Comment organiser une application pour séparer les responsabilités et faciliter la maintenance ?

---

## Bonnes pratiques essentielles

### Principe du MVC

Une application est découpée en 3 parties :

* Modèle (Model) :
  gère les données et la logique métier

* Vue (View) :
  gère l’affichage

* Contrôleur (Controller) :
  gère les interactions et les traitements

---

### Objectifs

* séparer les responsabilités
* rendre le code plus lisible
* faciliter les évolutions

---

### Fonctionnement

1. L’utilisateur interagit avec la vue
2. Le contrôleur traite la demande
3. Le modèle est sollicité
4. La vue est mise à jour

---

## Ce qu’il faut savoir pour le BTS

L’étudiant doit être capable de :

* expliquer le rôle de chaque composant
* identifier les éléments MVC dans une application
* comprendre l’intérêt de cette architecture
