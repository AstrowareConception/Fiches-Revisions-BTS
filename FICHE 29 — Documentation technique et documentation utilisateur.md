# FICHE 29 — Documentation technique et documentation utilisateur

---

## Problématique

Pourquoi documenter une application et comment adapter la documentation selon le destinataire ?

---

## Bonnes pratiques essentielles

### Rôle de la documentation

La documentation permet :

* de comprendre l’application ;
* de faciliter la maintenance ;
* d’aider les utilisateurs ;
* de transmettre des informations techniques.

Objectif :

* rendre le projet exploitable dans le temps.

---

### Documentation technique

Destinée aux développeurs et techniciens.

Elle peut contenir :

* architecture de l’application ;
* structure de la base de données ;
* API ;
* installation ;
* configuration ;
* procédures techniques.

Objectif :

* permettre la maintenance et les évolutions.

---

### Documentation utilisateur

Destinée aux utilisateurs finaux.

Elle doit expliquer :

* comment utiliser l’application ;
* les fonctionnalités principales ;
* les manipulations importantes.

Exemples :

* guide utilisateur ;
* tutoriel ;
* FAQ.

Objectif :

* rendre l’application facile à utiliser.

---

### Qualités d’une bonne documentation

Une documentation doit être :

* claire ;
* structurée ;
* à jour ;
* adaptée au destinataire.

---

### Captures et schémas

La documentation peut contenir :

* captures d’écran ;
* diagrammes ;
* schémas UML ;
* exemples.

Objectif :

* améliorer la compréhension.

---

### Mise à jour

Une documentation doit évoluer avec le projet :

* nouvelles fonctionnalités ;
* changements techniques ;
* corrections.

Une documentation obsolète devient inutile.

---

## Ce qu’il faut savoir pour le BTS

L’étudiant doit être capable de :

* distinguer documentation technique et utilisateur ;
* produire une documentation simple ;
* adapter son contenu au public visé ;
* expliquer l’intérêt de documenter un projet.

Le référentiel BTS insiste fortement sur la qualité de la documentation et de la communication technique.
