# FICHE 30 — Analyse des besoins et cahier des charges

---

## Problématique

Comment comprendre précisément les attentes d’un client avant de développer une application ?

---

## Bonnes pratiques essentielles

### Analyse des besoins

L’analyse des besoins consiste à :

* comprendre le problème du client ;
* identifier les fonctionnalités attendues ;
* déterminer les contraintes du projet.

Objectif :

* développer une solution adaptée au besoin réel.

---

### Cahier des charges

Le cahier des charges décrit :

* les objectifs du projet ;
* les fonctionnalités ;
* les contraintes techniques ;
* les contraintes juridiques ;
* les délais et attentes.

Objectif :

* servir de référence tout au long du projet.

---

### Besoins fonctionnels

Les besoins fonctionnels décrivent :

* ce que doit faire l’application.

Exemples :

* créer un compte ;
* gérer des commandes ;
* envoyer des messages.

---

### Besoins non fonctionnels

Les besoins non fonctionnels concernent :

* la sécurité ;
* les performances ;
* l’ergonomie ;
* la disponibilité ;
* la compatibilité.

Exemple :

* application accessible sur mobile ;
* temps de réponse inférieur à 2 secondes.

---

### Contraintes

Le projet peut comporter :

* contraintes techniques ;
* contraintes budgétaires ;
* contraintes réglementaires ;
* contraintes de sécurité.

Exemple :

* respect du RGPD ;
* utilisation d’un framework imposé.

---

### Importance de la reformulation

Le développeur doit :

* reformuler le besoin ;
* vérifier la compréhension ;
* éviter les ambiguïtés.

Objectif :

* limiter les erreurs de conception.

---

### Évolution des besoins

Un besoin peut évoluer :

* ajout de fonctionnalités ;
* changement métier ;
* nouvelles contraintes.

Le projet doit pouvoir s’adapter.

---

## Ce qu’il faut savoir pour le BTS

L’étudiant doit être capable de :

* analyser un besoin exprimé ;
* distinguer besoins fonctionnels et non fonctionnels ;
* exploiter un cahier des charges ;
* proposer une solution cohérente.

L’analyse du besoin est une compétence centrale du référentiel E5 SLAM.
