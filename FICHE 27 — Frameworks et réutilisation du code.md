# FICHE 27 — Frameworks et réutilisation du code

---

## Problématique

Pourquoi utiliser un framework dans une application et comment facilite-t-il le développement ?

---

## Bonnes pratiques essentielles

### Définition d’un framework

Un framework est un ensemble d’outils, bibliothèques et règles permettant de développer plus rapidement une application.

Objectifs :

* gagner du temps ;
* structurer le projet ;
* réutiliser du code existant.

---

### Exemples de frameworks

| Langage    | Framework        |
| ---------- | ---------------- |
| PHP        | Laravel, Symfony |
| Java       | Spring           |
| JavaScript | React, Vue.js    |
| Python     | Django           |

---

### Fonctionnalités apportées

Un framework fournit souvent :

* gestion des routes ;
* authentification ;
* accès base de données ;
* sécurité ;
* architecture MVC ;
* outils de tests.

---

### Avantages

Un framework permet :

* un code plus structuré ;
* une meilleure maintenabilité ;
* une sécurité renforcée ;
* une standardisation du développement.

---

### Réutilisation du code

Principe important :

* éviter de réécrire plusieurs fois le même traitement ;
* centraliser les fonctionnalités communes.

Exemples :

* composants ;
* services ;
* classes utilitaires.

Objectif :

* réduire les erreurs ;
* faciliter les évolutions.

---

### Bibliothèques et composants

Un projet peut intégrer :

* des bibliothèques externes ;
* des packages ;
* des composants réutilisables.

Exemple :

* gestion PDF ;
* authentification ;
* génération Excel.

---

### Maintenance et évolutions

Un framework facilite :

* les mises à jour ;
* la correction des bugs ;
* le travail collaboratif ;
* la documentation.

---

## Ce qu’il faut savoir pour le BTS

L’étudiant doit être capable de :

* expliquer l’intérêt d’un framework ;
* identifier les fonctionnalités fournies ;
* comprendre la notion de réutilisation du code ;
* justifier l’utilisation d’un framework dans un projet.

Le référentiel BTS insiste sur l’exploitation des frameworks et des composants logiciels.
